Question 1

1.a. How is the GET method different from POST when it comes HTML form submission?

The GET method sends the form data through the URL, while POST method sends the data through the body of the HTTP request, so the data is not shown in the URL.

1.b. Identify several factors/reasons (min of 3 each) for choosing one method (GET/POST) over the other.

GET
- Useful for when a user wants to bookmark the result
- Better when there is non-secure data, like google search
- Can be cached

POST
- Useful when there is sensitive data that you don't want users to know about it
- Has no size limitations (since the length of an url is limited, GET method has limit as well)
- Can upload files

1.c. What does it mean when one gets a HTTP status code in the range of 400-499? What about in the range of 500-599?

HTTP code in the range of 500-599 is when the server is awared it has erred os is incapable of performing the request. HTTP code in the range of 400-499 is for cases in which the client seems to have erred.

1.d. What does HTTP status code 403 mean? What about 404? What about 500?

403 means forbidden, the server understood the request but is can't send an answer. 404 means not found, the server has not found anything matching the URL. 500 means internal server error, the server encountered an unexpected condition which prevent it for finishin the request.
